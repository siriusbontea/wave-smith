## What Wavesmith is

Local-first AI music studio (Suno-style): describe a song ‚Üí forge full tracks with vocals ‚Üí library ‚Üí stems. **100% on the user's machine** ‚Äî no accounts, cloud APIs, or telemetry. **GPL-3.0-or-later.**

**Verified platform:** macOS Apple Silicon. Linux/CUDA: [docs/LINUX_CUDA.md](docs/LINUX_CUDA.md) (documented only, not tested).

### Current state

Milestones **M0‚ÄìM7 are implemented** on `master`: engine ground truth, app skeleton, generate queue, Create UI, Library/players, Demucs stems, settings/tour, tests + `scripts/smoke.sh` + README.

Before claiming a feature works, run the relevant gate:

```bash
MOCK_ENGINE=1 pnpm test          # must pass with NO engine
MOCK_ENGINE=1 pnpm build         # strict tsc clean
MOCK_ENGINE=1 pnpm test:e2e      # Playwright smoke
./scripts/smoke.sh               # real engine (optional, slow)
```

## Architecture (locked ‚Äî do not redesign)

```
Browser ‚Üí Wavesmith (Next.js @ :3000) ‚Üí ACE-Step REST API (@ :8001)
              ‚îú‚îÄ‚îÄ SQLite (library, jobs, settings)
              ‚îú‚îÄ‚îÄ ./data/audio
              ‚îú‚îÄ‚îÄ Demucs CLI (stems)
              ‚îî‚îÄ‚îÄ Ollama (optional lyrics LLM)
```

### Processes

| Process | Port | Notes |
|---------|------|-------|
| **Wavesmith app** | 3000 | Next.js 16 App Router; route handlers *are* the API; bind `127.0.0.1` only |
| **ACE-Step 1.5** | 8001 | Cloned to `engine/ACE-Step-1.5` (gitignored), pin in `docs/ENGINE_NOTES.md`. REST only ‚Äî never Gradio, never custom PyTorch |
| **Ollama** (optional) | 11434 | Lyrics LLM; degrades gracefully when absent |

### Invariants (never break these)

1. **One audio playback owner** ‚Äî single global `HTMLAudioElement` in `lib/audio/store.ts` (Zustand). wavesurfer.js v7 uses MediaElement mode on that same element. No second player.
2. **Job queue singleton** ‚Äî `lib/queue/`, cached on `globalThis` (HMR-safe). Concurrency **1** toward the engine. Persist to `jobs` table; on boot re-enqueue `queued`, fail orphaned `running`.
3. **Polling, not SSE** ‚Äî TanStack Query ~1s `refetchInterval` while jobs are active.
4. **Audio Range support** ‚Äî `app/api/audio/[...path]/route.ts` must return **206** for seeks; validate paths against traversal.
5. **Stems via Demucs** ‚Äî `htdemucs` CLI, not engine Extract (turbo variant loaded; `base` needed for engine stems).
6. **Lyrics passthrough** ‚Äî no app-side censorship; user lyrics go to the engine unmodified.
7. **Test seam** ‚Äî `MOCK_ENGINE=1` ‚Üí `MockEngineClient`; tests must never require GPU or ACE-Step running.

### Engine & lyrics seams

- `lib/engine/` ‚Äî `EngineClient` + `AceStepClient` + `MockEngineClient`
- `lib/lyrics/` ‚Äî `LyricsClient` + `OllamaLyricsClient` + `MockLyricsClient`

Stub failing externals behind these interfaces; log in [docs/ESCAPE_HATCHES.md](docs/ESCAPE_HATCHES.md).

## Commands

| Command | Purpose |
|---------|---------|
| `./scripts/setup.sh` | First-time / idempotent setup (engine clone, ~10 GB weights, demucs, pnpm, migrate) |
| `./scripts/dev.sh` | **Daily dev** ‚Äî engine + `next dev`; open **http://127.0.0.1:3000** |
| `./scripts/start.sh` | Production build + start |
| `./scripts/smoke.sh` | E2E vs real engine |
| `pnpm build` | Production build; must be clean |
| `pnpm typecheck` | Strict `tsc` |
| `pnpm test` | Vitest ‚Äî use `MOCK_ENGINE=1` |
| `pnpm test:e2e` | Playwright ‚Äî use `MOCK_ENGINE=1` |

**Common mistake:** Forge 404 means the **app on :3000** is down, not the engine on :8001. Both are started by `dev.sh`.

## Code conventions

- **TypeScript strict**, no `any`
- **Zod** every route handler input (`.strict()` schemas)
- **Header comment** on non-trivial modules: role + data flow; inline comments explain *why*
- **Judgment calls** ‚Üí one line in [docs/DECISIONS.md](docs/DECISIONS.md)
- **README numbers** ‚Üí measured on this machine only (see `docs/ENGINE_NOTES.md` ¬ß9)
- **Commits** ‚Üí conventional messages; only when the user asks (unless their rules say otherwise)
- **Do not redesign** locked architecture; engine API changes require updating `docs/ENGINE_NOTES.md` first

### Data model (Drizzle ‚Äî four tables)

`songs` ¬∑ `jobs` (`generate` \| `stems`) ¬∑ `stems` ¬∑ `settings` ‚Äî see [db/schema.ts](db/schema.ts). `songs.lrc` and `songs.quality_score` stay null (REST API does not expose them; M0-verified).

## What agents should do

1. Read relevant files before editing; match existing style and patterns.
2. Prefer minimal, focused diffs ‚Äî extend seams (`EngineClient`, `LyricsClient`), don't bypass them.
3. Run `MOCK_ENGINE=1 pnpm test` and `pnpm build` after non-trivial changes.
4. Record non-obvious decisions in `docs/DECISIONS.md`.
5. Keep GPL license notices intact; ACE-Step remains a separate MIT upstream (not vendored).

## What agents should not do

- Add a second audio engine, Redis, separate backend service, or cloud dependencies
- Commit `.env`, `data/`, `engine/`, or model weights
- Use Gradio or engine Python APIs directly from the app
- Censor user-supplied lyrics in app code
- Attribute AI tools in commits, comments, or packaging
- Present upstream marketing performance numbers as measured facts
